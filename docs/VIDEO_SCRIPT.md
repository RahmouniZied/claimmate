# Video script (under 2 minutes, public, no login)

Record the live app (`adk web`) with one short cut to the trace view. All data is synthetic.

| Time | On screen | Voiceover |
|---|---|---|
| 0:00 to 0:12 | A denial letter | "Your insurer just denied this claim. They are betting you will not appeal, because the deadline is buried and the policy is impossible to read." |
| 0:12 to 0:30 | Drop the letter into ClaimMate. It states the facts and the deadline. | "ClaimMate reads the letter, checks the diagnosis code against a public medical database, and finds the deadline you would have missed." |
| 0:30 to 0:55 | Split screen. Left: a plain model invents a clause number. Right: ClaimMate cites a real clause, and on a second case says it cannot find support. | "A normal assistant will invent a policy clause that does not exist. ClaimMate cites the real clause, or it tells you the truth: no supporting language, and it will not guess." |
| 0:55 to 1:12 | The orchestrator calls the Clause-Finder. Show the agent card or the tool call. | "A specialist agent does the hard reading and decides whether the plan actually supports you. They talk over the agent-to-agent protocol." |
| 1:12 to 1:30 | The appeal assembles, every citation traceable. The citation gate passes. | "It drafts your appeal using only real clauses. A gate blocks anything invented before you ever see it." |
| 1:30 to 1:42 | The draft is ready. The approval prompt appears. | "And it stops here. It will not send a single word without your approval." |
| 1:42 to 1:52 | New session weeks later. It already knows the plan. | "Weeks later, a new denial. It already remembers your plan, so you start where you left off." |
| 1:52 to 2:00 | Quick cut to the trace view and the eval pass-rate. | "Real tools, real grounding, measured. ClaimMate finds the clause they ignored, and never sends a word without you." |
